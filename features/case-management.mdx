## Overview

Juryo's case management system helps law firms organize and track legal matters from initial consultation through case closure. Cases represent active legal work, with document tracking, attorney assignment, status workflows, and complete activity history.

<Info>
**Key Capabilities**
- Customizable case status workflows
- Document management with AI review
- Attorney assignment and collaboration
- Requirement tracking per legal process
- Complete audit trail of all activities
- Payment plan integration
</Info>

## Case Lifecycle

Cases progress through customizable statuses as legal work advances.

### Standard Status Flow

| Status | Description | Typical Duration |
|--------|-------------|------------------|
| **Consultation** | Initial case review and assessment | 1-7 days |
| **Active** | Work in progress | Varies by case type |
| **Waiting on Client** | Awaiting client documents or payment | Variable |
| **Waiting on Third Party** | Pending external responses (USCIS, courts, etc.) | Variable |
| **Under Review** | Internal review before submission | 3-7 days |
| **Submitted** | Filed with agency or court | Variable |
| **Completed** | Case successfully closed | N/A |
| **Cancelled** | Case terminated before completion | N/A |

<Note>
Case statuses are customizable per tenant. Admins can configure workflows specific to their practice areas.
</Note>

## Creating Cases

### From Won Opportunity

The most common way to create a case is by converting a won opportunity:

1. Open the opportunity (must be in "Won" stage)
2. Click **Convert to Case**
3. System pre-fills information:
   - **Client**: Opportunity's contact (automatically updated to "Client" role)
   - **Legal Area**: From opportunity service type
   - **Legal Process**: From opportunity details
   - **Case Value**: From opportunity estimated value
4. Add case-specific details:
   - Case name/number (auto-generated or custom)
   - Assigned attorney
   - Initial status (typically "Consultation")
   - Expected completion date
   - Case notes
5. Click **Create Case**

**Automatic Actions:**
- Contact role updated to "Client"
- Opportunity linked to case
- Communication history transferred
- Document requirements populated (based on legal process)
- Initial tasks created (if workflow configured)

### Manual Case Creation

Create a case directly without an opportunity:

1. Navigate to **Cases** page
2. Click **New Case**
3. Select or create client contact
4. Fill in case details:
   - **Case Name**: Descriptive title
   - **Legal Area**: Practice area (Immigration, Corporate, Family, etc.)
   - **Legal Process**: Specific procedure (H-1B Visa, LLC Formation, etc.)
   - **Assigned Attorney**: Primary team member
   - **Status**: Initial status
   - **Priority**: High, Medium, Low
   - **Fee Structure**: Flat fee, hourly, contingency
   - **Case Value**: Expected revenue
5. Click **Create Case**

### API Case Creation

```javascript
const response = await fetch('/api/cases', {
  method: 'POST',
  credentials: 'include',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    clientId: "contact-uuid",
    caseName: "H-1B Visa - Software Engineer",
    legalAreaId: "immigration-uuid",
    legalProcessId: "h1b-visa-uuid",
    assignedAttorneyId: "attorney-uuid",
    status: "consultation",
    priority: "high",
    estimatedValue: 8000,
    notes: "Initial consultation scheduled for Jan 15"
  })
});

const { data } = await response.json();
console.log('Case created:', data.id);
```

See [Create Case API](/api-reference/cases/create-case) for full documentation.

## Document Requirements

### Requirement Templates

Legal processes have predefined document requirements:

**Example: H-1B Visa Requirements**
```json
{
  "legalProcess": "H-1B Visa",
  "requirements": [
    {
      "name": "Valid Passport",
      "type": "specific_document",
      "required": true,
      "aiReviewable": true,
      "description": "Passport must be valid for at least 6 months beyond intended stay"
    },
    {
      "name": "Bachelor's Degree Diploma",
      "type": "specific_document",
      "required": true,
      "aiReviewable": true
    },
    {
      "name": "Employment Offer Letter",
      "type": "specific_document",
      "required": true,
      "aiReviewable": false
    },
    {
      "name": "Resume/CV",
      "type": "general_document",
      "required": true
    }
  ]
}
```

### AI Document Review

For requirements marked as `aiReviewable: true`, Juryo can automatically validate documents:

**Review Process:**
1. Document uploaded and linked to requirement
2. Click **Request AI Review**
3. Google Gemini AI analyzes document
4. System returns:
   - **Status**: Approved or Rejected
   - **Confidence**: 0-100% score
   - **Feedback**: Specific issues or confirmation
5. Requirement status updated automatically

**Example AI Review:**
```json
{
  "documentId": "doc-uuid",
  "requirementId": "req-uuid",
  "status": "approved",
  "confidence": 92,
  "feedback": "Passport is valid and meets all requirements. Expiration date: 2028-05-15. Valid for 6+ months beyond intended stay.",
  "reviewedAt": "2025-01-08T15:30:00Z"
}
```

<Warning>
**AI Review Limitations**

AI document review is a tool to assist attorneys, not replace human review. Always verify AI decisions, especially for critical documents.
</Warning>

See [AI Document Review](/integrations/ai-document-review) for setup and capabilities.

## Case Status Management

### Updating Status

**Manual Update:**
1. Open case
2. Click status dropdown
3. Select new status
4. Optionally add note explaining change
5. Click **Update**

**Automatic Update:**
Status changes can trigger automatically based on:
- Task completions
- Document uploads
- Payment received
- External webhook events
- Time-based rules

**Example Automation:**
```
Trigger: All required documents marked "Completed"
AND Payment plan status is "Paid"
Action: Update case status to "Under Review"
```

### Status Change Notifications

When case status changes, notifications can be sent to:
- Assigned attorney
- Full case team
- Client (via email or WhatsApp)
- External systems (via webhook)

**Notification Configuration:**
Configure per status which parties should be notified and through which channels.

### Creating Payment Plans

Link payment plans to cases for installment billing:

1. Open case
2. Navigate to **Payments** section
3. Click **Create Payment Plan**
4. Configure:
   - Total amount
   - Number of installments
   - Due dates
   - Payment method
5. Click **Create**

**Payment Statuses:**
- **Pending**: Not yet paid
- **Partially Paid**: Some installments paid
- **Paid**: Fully paid

- **Cancelled**: Payment plan cancelled

**Automation Integration:**

## Case Closure

### Completing Cases

When legal work is finished:

1. Update status to "Completed"
2. Confirm all requirements met
3. Ensure all documents finalized
4. Verify payment received
5. Add completion notes
6. Optionally generate completion report

**Automatic Actions on Completion:**
- Client role optionally updated to "Former Client"
- Final invoice generated (if applicable)
- Completion notification sent
- Case archived after retention period

### Cancelling Cases

If case cannot proceed:

1. Update status to "Cancelled"
2. Select cancellation reason:
   - Client withdrew
   - Payment issues
   - Conflict of interest
   - Transferred to another firm
   - Ineligible for service
3. Add cancellation notes
4. Process refund (if applicable)

**Cancelled Case Handling:**
- Remains in system for records
- Communication history preserved
- Documents retained per policy
- Not included in active case metrics

## Best Practices

<AccordionGroup>
  <Accordion title="Update case status regularly">
    **Why:** Accurate status tracking enables better resource planning and client communication.

    **How:** Review cases weekly. Update status as work progresses. Use automation to trigger status changes based on milestones.
  </Accordion>

  <Accordion title="Link all communications to cases">
    **Why:** Complete case history is essential for continuity and quality service.

    **How:** Use WhatsApp/voice integrations for automatic linking. Manually link emails and calls to relevant cases.
  </Accordion>

  <Accordion title="Track requirements proactively">
    **Why:** Missing documents cause delays. Proactive tracking keeps cases moving forward.

    **How:** Review requirements tab weekly. Send reminders for pending documents. Use automation for follow-ups.
  </Accordion>

  <Accordion title="Assign cases appropriately">
    **Why:** Proper workload distribution prevents burnout and maintains quality.

    **How:** Monitor case load per attorney. Consider complexity and urgency when assigning. Balance across team.
  </Accordion>

  <Accordion title="Use AI review as a first pass">
    **Why:** AI review catches obvious issues quickly, allowing attorneys to focus on complex analysis.

    **How:** Request AI review immediately after document upload. Have attorney verify AI decisions. Document any overrides.
  </Accordion>
</AccordionGroup>
